Git - Working With Branches

Skip to end of metadata
  • Created by Scott Leon, last modified on Mar 17, 2014
Go to start of metadata

Based on the "A successful Git branching model" post

Creating a branch (from a parent branch)

?
$ git checkout -b <branch name> <parent branch>
Switched to a new branch "<branch name>"

 

Rebasing a branch (from a parent branch)

Note: If in the "push" step, Git complains, see your team lead for next steps

?
$ git checkout <parent branch>
$ git pull
$ git checkout <branch name>
$ git pull
$ git rebase <parent branch>
$ git push <parent branch>

 

Merging a branch (to a parent branch)

Note: If you're merging a feature branch, make sure that:

  1. The feature branch has passed QA testing
  2. The feature branch has been cleared to be added to the next release
  3. The feature branch has first been rebased (from develop)

Note: Only team leads should be merging feature branches to develop

Note: Only team leads should be merging release candidate branches to master 

?
$ git checkout <parent branch>
$ git pull
$ git merge --no-ff <branch name>
Note: There will likely be conflicts mentioned by Git
Note: If you have any questions about resolving one or more conflicts, see your team lead for next steps
$ git push origin <parent branch>

 

Deleting a branch

Note: This step should only be performed by the creator of the branch

?
$ git branch -d FE-<feature name>

 

Cherry picking branch commits (to parent branch)

Note: See Matt M for comparison script

?
$ git checkout <parent branch>
$ git cherry-pick <oldest commit hash> <next oldest commit hash> .. <newest commit hash>
